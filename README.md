# GeekAdvice.au

The personal site of Peteyyy — a hand-built replacement for the old WordPress install at
[geekadvice.au](https://geekadvice.au). Built with **Next.js + MDX**, hosted on
**Azure Static Web Apps**, content version-controlled in **Git**.

> Home of Race Wars, music, coding, Mac gaming, and the occasional thing-that-broke-at-2am.

---

## What's in the box

- **Homepage** (`/`) — hero, featured latest post, three more recent posts, three section cards
  (Music / MacOS Gaming / About).
- **Blog index** (`/blog/`) — every post, with click-to-filter tag pills.
- **Blog post pages** (`/blog/<slug>/`) — MDX content with syntax highlighting, prev/next
  navigation, tag pills.
- **Tag pages** (`/tags/<tag>/`) — generated at build time, one per tag.
- **About** (`/about/`), **Music** (`/music/` with embedded SoundCloud player),
  **MacOS Gaming** (`/macos-gaming/`).
- **RSS feed** at `/rss.xml` (latest 20 posts, full content).
- **404 page** styled to match — one global, one for unknown blog slugs.
- **SEO**: per-page metadata, Open Graph, sitemap, robots.txt, favicon, OG image.
- **Security headers** via `staticwebapp.config.json`: HSTS, X-Content-Type-Options,
  Referrer-Policy, X-Frame-Options, Permissions-Policy, plus 301 redirects from the old
  `/index.php/*` WordPress URLs to the new ones.

### Design

- **Pastel 90s palette** (white base, pastel purple/green/yellow/blue/pink) wired as CSS custom
  properties in `src/app/globals.css` and exposed as Tailwind utilities.
- **Milkshake splatter background** — fixed radial-gradient blobs in seven spots, scrolls with
  the viewport.
- **Typography**: Inter for body, **VT323** for retro accent headings (used sparingly on h1/h2),
  JetBrains Mono for code and nav.
- **Cards** — rounded 2xl with subtle shadow, lift on hover, the entire card is clickable via a
  stretched-link pattern; inner tag pills and the "read more" link remain independently
  clickable.
- **Accessible**: skip-link, focus rings, `prefers-reduced-motion` honoured, semantic HTML,
  AA-contrast text on accent backgrounds.

### Why this stack

- **No database, no API server.** No PHP, no SQL injection surface, no WordPress plugin CVEs.
  Posts are `.mdx` files in Git.
- **No runtime.** `next build` produces a fully static `out/` directory served from a CDN.
  Impossible to hack in the traditional sense.
- **Git as the CMS.** Write a post, `git push`, live in ~60 seconds. PRs get preview URLs.

## Authoring a new post

Create `src/content/posts/<date>-<slug>.mdx` (the `YYYY-MM-DD-` prefix is just for sorting in
your file tree — it's stripped from the URL).

```mdx
---
title: "My new post"
slug: "my-new-post"        # optional, defaults to filename
date: 18-06-2026           # dd-mm-yyyy
description: "Short summary for SEO and the post card."
tags: ["race-wars", "gamedev"]
cover: "/images/posts/my-new-post.png"   # optional
draft: false               # true = excluded from build
---

Your content here. Standard markdown plus React components:

<Callout type="tip">Heads up — this is a callout.</Callout>

<YouTube id="dQw4w9WgXcQ" title="Embedded video" />

<OutLink href="https://example.com">External link</OutLink>
```

- Custom components available inside posts: `<Callout>` (info/warn/tip), `<YouTube>`, `<OutLink>`.
- `draft: true` excludes the post from `/blog`, tag pages, and RSS without deleting the file.
- Dates support both `dd-mm-yyyy` (preferred) and ISO `yyyy-mm-dd` in the YAML.
- Save and `git push` — the post is live in about a minute.

### Authoring pages

The three fixed content pages (`/about/`, `/music/`, `/macos-gaming/`) live in
`src/content/pages/`. Same frontmatter format, but they aren't posts — they don't appear in the
blog index or RSS.

## Deploying to Azure Static Web Apps

### One-time setup

1. **Create a new GitHub repo** and push this project.

   ```bash
   git init
   git add -A
   git commit -m "initial scaffold"
   git branch -M main
   git remote add origin git@github.com:<you>/geekadvice.au.git
   git push -u origin main
   ```

2. **Create the Azure Static Web App** in the [Azure Portal](https://portal.azure.com):
   - **Resource Group**: your existing one (e.g. `geekadvice-prod`)
   - **Name**: `geekadvice-au` (becomes the default hostname `geekadvice-au.azurestaticapps.net`)
   - **Plan type**: **Free**
   - **Region**: **Australia East** (closest to your audience)
   - **Source**: **GitHub** → sign in → pick the repo + branch `main`
   - **Build Preset**: **Next.js**
   - **App location**: `/`
   - **Output location**: `out`

   Azure auto-creates `.github/workflows/azure-static-web-apps.yml` in your repo and triggers
   the first build (~2 min).

3. **Add the custom domain**:
   - In the SWA resource → **Custom domains** → **Add** → `geekadvice.au`
   - Azure gives you a `TXT` validation record and an `A` (or `CNAME`) target record.
   - At your current registrar (where `geekadvice.au` is registered):
     - Add the `TXT` validation record at `@`.
     - Add the `A` record at `@` pointing at the Azure IP (apex domains can't CNAME; SWA will
       provide the IP).
     - Add a `CNAME` record at `www` pointing at your `*.azurestaticapps.net` hostname.
   - Wait for validation (~5 min) and free SSL cert auto-provision.
   - Optionally enable HTTPS-only and the apex→www or www→apex redirect.

4. **Smoke test** the live site:
   - Visit `https://geekadvice.au/`, `/blog/`, `/about/`, `/rss.xml`.
   - Confirm the old `https://geekadvice.au/index.php/sample-page/` redirects to `/about/` via
     the rules in `staticwebapp.config.json`.
   - In Lighthouse (DevTools), confirm Performance / A11y / Best Practices / SEO all ≥ 95.

5. **Decommission WordPress** (the security rationale for this rebuild):
   - Take a final MySQL/database backup.
   - Stop the VM / App Service running WordPress.
   - Keep the backup offline for 30 days in case you need to recover old content, then delete.

### Day-to-day authoring workflow

```bash
# 1. Write a new post
$EDITOR src/content/posts/18-06-2026-my-new-post.mdx

# 2. Commit & push
git add src/content/posts/18-06-2026-my-new-post.mdx
git commit -m "post: my new post"
git push origin main
```

GitHub Actions rebuilds and SWA deploys in ~60 seconds. The Azure bot will comment on the commit
with the live URL.

### PR previews

Every pull request automatically gets a **preview URL** at
`https://<random>-<swa-name>.azurestaticapps.net`. Useful for showing a draft post to a friend
before merging to `main`.

## Customising

- **Site name, tagline, social links** — `src/lib/site.ts`.
- **Menu** — `site.nav` in the same file. The first item ("Home") is treated as the wordmark.
- **Palette** — CSS custom properties at the top of `src/app/globals.css`. Tailwind utilities
  `bg-purple`, `text-green`, etc. read from these.
- **Hero copy / homepage tagline** — `src/app/page.tsx`.
- **Security headers & WP redirects** — `staticwebapp.config.json`.
- **Footer version line** — `src/components/site/SiteFooter.tsx`.

## Operational notes

- **First Load JS**: ~94 KB per page (87 KB shared + page-specific). Well within Lighthouse
  green.
- **Total `out/` size**: ~2.1 MB across 24 HTML files + assets.
- **Free tier limits**: 100 GB bandwidth/month, 0.5 GB storage, custom domain + free SSL, PR
  previews. Standard plan (~$9/mo) is the upgrade path if you ever blow past that.
- **Backups**: the Git repo IS the backup. Every post is version-controlled. No separate
  database to back up.
